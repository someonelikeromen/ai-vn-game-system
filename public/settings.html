🔗 接口与模型（统一配置）

主对话与商城模型集中配置。新手只需先填主接口，商城可按需覆盖。

推荐流程：先配置主接口（BaseURL / API Key / 模型）并测试；商城接口留空即可继承，只有需要分流模型时再单独填写。

🏪 商城接口覆盖（可选）

留空表示继承主接口，适合“傻瓜模式”；仅在你想让商城走独立模型时填写。

提示：商城使用独立的 LLM 来生成道具、同伴和增益效果。建议使用响应速度较快的模型。

⚡ 多阶段推理

将每次发送拆分为规划→检索→生成→更新四个阶段，提升叙述质量。

阶段1·规划：草拟大纲、标记需要的上下文  |  阶段2·检索：RAG 检索历史与战斗数据  |  阶段3·生成：正文叙述  |  阶段4·更新：生成状态更新语句
阶段1 · 规划 留空字段继承主接口
阶段3 · 生成叙事 留空字段继承主接口
阶段4 · 状态更新 留空字段继承主接口
